# SIPF
Smart Intrusion Prevention Firewall (SIPF): Real-Time Defense against SYN Port Scans and SYN Flood Attacks
Networks today have been subjected to unrelenting reconnaissance and denial-of-service (DoS) attacks that take advantage of flaws in the TCP handshake protocol. This report is a project proposal on Smart Intrusion Prevention Firewall (SIPF) which is a real time network-level defense mechanism that can automatically identify and prevent TCP SYN port scans and TCP SYN flood DoS attacks. SIPF follows abnormal SYN flag patterns and connection rate in the inbound TCP traffic. When the suspicious activity is detected (e.g. when a user makes a large number of SYNs to multiple ports or when a large number of half-open connections are detected), new firewall rules (added by iptables) are dynamically injected to block or rate-limit the rogue hosts in real-time. The rules are subsequently eliminated following a safe cool off period to prevent the occurrence of false positives in the long term. The SIPF is an intelligent packet analyzer and an automation-based system using Python to analyze packets and react to them faster than manually crafted rules, which is configured with the Linux firewall
